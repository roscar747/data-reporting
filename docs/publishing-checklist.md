# Publishing checklist

Run through this list before pushing a project to a public branch. It is deliberately short and binary — every item is either a yes or it blocks the push.

## Repository hygiene

- [ ] `git status` is clean except for the changes you intend to publish
- [ ] No files larger than 100 MB outside of Git LFS
- [ ] `.gitignore` excludes `**/.pbi/localSettings.json`, `**/.pbi/cache.abf`, `*.pbids`, `secrets.json`, `.env*` (except `.env.example`)
- [ ] `.gitattributes` is tracking `*.pbix`, `*.pbit`, `*.parquet` through LFS
- [ ] No internal hostnames, IP addresses, or private URLs in any committed file (`grep -rE "(\.local|\.internal|10\.[0-9]|192\.168\.)" .` returns nothing relevant)
- [ ] `LICENSE` file is present at the repo root

## Per-project content

- [ ] `README.md` exists and follows the [project README template](../templates/project-readme-template.md)
- [ ] Business problem statement is written for a non-technical reader (no acronyms left undefined)
- [ ] Dataset description names every table and gives row counts
- [ ] At least one hero screenshot in `reports/exports/screenshots/preview.png`
- [ ] Per-page screenshots numbered `01-…`, `02-…`, etc.
- [ ] PDF export is in `reports/exports/pdf/<BusinessName>_v<n>.pdf`
- [ ] Wireframe is paired with a final-dashboard screenshot in the README's *Wireframe vs final* section
- [ ] [`docs/data-dictionary.md`](../templates/data-dictionary-template.md) is filled in for every table the report consumes
- [ ] [`docs/kpi-definitions.md`](../templates/kpi-definitions-template.md) is filled in for every measure surfaced in a visual

## Power BI artifacts

- [ ] `.pbix` opens without errors using only the data files in this repo (no missing source warnings)
- [ ] *Home → Transform data → Data source settings* contains no references to private data sources
- [ ] *Modeling → Manage roles* contains no internal emails / AD group names
- [ ] *View → Performance analyzer* doesn't show any single visual taking >2 s on cold open
- [ ] Filter pane and bookmark/selection panes are hidden in the saved view (cleaner first impression)
- [ ] Report theme is applied consistently across all pages
- [ ] Page navigator / left-nav reflects the actual page list (no orphans)

## Synthetic data declaration

- [ ] Every CSV and dataset in `data/raw/` is fabricated, generated by script, or licensed for redistribution
- [ ] CSVs containing fabricated data have a `synthetic_`, `mock_`, or `dummy_` prefix in the filename — or the README's *Data &amp; privacy* section names them explicitly
- [ ] No real names, addresses, NHS numbers, account numbers, IP addresses, or geolocation traces in any sample
- [ ] The project README's *Data &amp; privacy* section reads "All data in this project is synthetic" or equivalent

## GitHub presentation

- [ ] Repo description is filled in (top-right of the GitHub page)
- [ ] Topics include `power-bi`, `data-analytics`, `dax`, `power-query`, plus a domain tag (`fintech`, `healthcare`, etc.)
- [ ] Pinned repos on your GitHub profile include this one
- [ ] Repo `About` section links to your LinkedIn or portfolio site
- [ ] Repo `README.md` renders correctly (open it in incognito to check images load over public URLs)

## Optional but recommended

- [ ] An animated GIF walkthrough in `reports/exports/screenshots/walkthrough.gif` for the headline project
- [ ] A "Featured dashboards" table at the top of the root README with three preview tiles
- [ ] A short Loom or YouTube video walkthrough linked from the project README
- [ ] A `CHANGELOG.md` per project capturing what changed between `_v1` and `_v2`
