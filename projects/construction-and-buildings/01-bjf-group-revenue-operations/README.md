## Business problem

Construction groups operate dozens of concurrent projects, each with its own commercial team, schedule and risk profile. The board sees a monthly roll-up that's already three weeks stale; the project managers see weekly site reports that don't roll up. The gap is a single shared semantic model with consistent definitions of *revenue*, *margin* and *progress* that both audiences can drill from and zoom out to.

This is BJF Group's revenue and operations report — the first project in the portfolio that ships in **Power BI Project (`.pbip`) format alongside the `.pbix`**. Reviewers can read the model definition in plain TMDL and the report definition in JSON without opening Power BI Desktop.

## Dataset description

The dataset for this project is **embedded in the semantic model** rather than served from a separate CSV. The relevant tables, defined in TMDL, are:

| Table | Grain | Source |
|-------|-------|--------|
| `construction_projects` | One row per project-month | `BJF_Group_Revenue_Operation_Report.SemanticModel/definition/tables/construction_projects.tmdl` |
| `DimDate` | One row per calendar date | `BJF_Group_Revenue_Operation_Report.SemanticModel/definition/tables/DimDate.tmdl` |
| `KPIs` | Pre-aggregated KPI table for executive summary | `BJF_Group_Revenue_Operation_Report.SemanticModel/definition/tables/KPIs.tmdl` |
| `_Time Intelligence` | Calculation group — MTD, QTD, YTD, PY, YoY% | `BJF_Group_Revenue_Operation_Report.SemanticModel/definition/tables/_Time Intelligence.tmdl` |

**Source.** Synthetic — fabricated to model a typical UK regional construction group with 15–25 concurrent projects across residential, commercial and infrastructure.

Full field definitions: [`docs/data-dictionary.md`](docs/data-dictionary.md).

## Key KPIs and insights

| KPI | What it answers | Target / threshold | Where it appears |
|-----|------------------|-------|------------------|
| Revenue (£) | Total invoiced + accrued revenue | n/a | Overview, Project |
| Gross margin % | Revenue minus direct cost, as % | ≥ 18% target | Overview |
| Schedule variance (days) | Days behind / ahead of baseline | Within ±10 days | Project |
| Cost-to-complete (£) | Forecast remaining cost on each open project | Watch vs budget | Project |
| Cash-flow cover | Forecast inflow vs committed outflow over 90 days | ≥ 1.1× | Cash flow |

The full set of measures, including time-intelligence variants generated by the `_Time Intelligence` calculation group, is documented in [`docs/kpi-definitions.md`](docs/kpi-definitions.md).

**Headline insights** (derived from the synthetic portfolio).

1. Two projects account for the majority of negative schedule variance, but they are profitable; the dashboard surfaces that schedule slip and margin slip don't move together.
2. Gross margin sits comfortably above target on residential, marginally above on commercial, and below on infrastructure — a portfolio mix conversation rather than a delivery conversation.
3. Cash-flow cover dips below 1.1× for two months mid-window. The dashboard's 90-day projection visual makes that obvious before it happens.

## Data &amp; privacy

All data is **synthetic**. The "BJF Group" name, projects, and financials are fabricated for the purposes of this portfolio.

## Tech notes

- **Semantic model.** Star schema with a `construction_projects` fact, a `DimDate` dimension, an aggregated `KPIs` table for executive summary cards, and a `_Time Intelligence` calculation group exposing MTD / QTD / YTD / PY / YoY% across every base measure.
- **Locale.** Modelled in `en-GB` (`BJF_Group_Revenue_Operation_Report.SemanticModel/definition/cultures/en-GB.tmdl`) — currency formats and date formats are British.
- **Theme.** Custom theme based on `CY24SU10`, with the BJF logo embedded under `BJF_Group_Revenue_Operation_Report.Report/StaticResources/RegisteredResources/`.
- **Local-date tables.** Multiple `LocalDateTable_…` tables exist as a side-effect of auto-generated relationships; these can be removed by disabling *Auto date/time for new files* in Power BI options if you fork this project.
